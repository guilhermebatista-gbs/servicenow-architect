# ServiceNow Fluent — Project Structure

## Standard Layout

```
my-app/
├── now.config.json             # Application scope config (scope, scopeId) — required
├── package.json                # NPM config — devDependencies: @servicenow/sdk, @servicenow/glide, typescript
├── tsconfig.json               # TypeScript config — types provided by @servicenow/sdk
├── pnpm-lock.yaml              # pnpm lockfile
└── src/
    ├── fluent/                 # All Fluent metadata definitions live here
    │   ├── my-table.now.ts     # One metadata definition per file (convention, not enforced)
    │   ├── my-acl.now.ts
    │   ├── my-rule.server.js   # External server scripts referenced via Now.include
    │   ├── my-script.client.js # External client scripts referenced via Now.include
    │   └── generated/          # Auto-generated by now-sdk build — never edit manually
    └── server/                 # Reusable JavaScript/TypeScript modules (business logic)
        ├── handler.ts          # A module exporting reusable functions
        └── utils.ts
```

## File Naming Conventions

| Extension | Purpose |
|-----------|---------|
| `.now.ts` | Fluent metadata definitions — compiled by `now-sdk build` |
| `.server.js` | Server-side scripts included via `Now.include` (co-located with `.now.ts`) |
| `.client.js` | Client-side scripts included via `Now.include` — **MUST use `.client.js` for ClientScript** |
| `.html` | HTML for UiPage definitions included via `Now.include` |
| `.ts` (in `src/server/`) | Reusable TypeScript modules — compiled to `dist/modules/server/` |

> **ClientScript files MUST use the `.client.js` extension.** Using `.server.js` for a ClientScript is an error.

## Scope Prefix Rule

> **CRITICAL: Scope prefix MUST be read from `now.config.json`. Never invent or assume the scope prefix.**

The `scope` field in `now.config.json` is the single source of truth for the application scope prefix. All table names, role names, and API names in your `.now.ts` files must use the exact scope prefix from this file.

```json
// now.config.json
{
  "scope": "x_snc_example_app",
  ...
}
```

Correct usage:
```typescript
// Table name uses scope prefix from now.config.json
export const x_snc_example_app_incident = Table({ name: 'x_snc_example_app_incident', ... })

// Role name uses scope prefix
export const adminRole = Role({ name: 'x_snc_example_app.admin' })

// ScriptInclude apiName uses scope prefix
ScriptInclude({ apiName: 'x_snc_example_app.MyUtility', ... })
```

## now.config.json (full schema)

```json
{
  "scope": "x_snc_example_app",
  "scopeId": "2f6400eb87426118f736e28f69d3017a",
  "name": "ExampleApp",
  "tsconfigPath": "./src/server/tsconfig.json"
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `scope` | **Yes** | Application scope prefix (e.g. `x_snc_example_app`) — used to prefix all metadata |
| `scopeId` | **Yes** | sys_id of the application in ServiceNow |
| `name` | No | Application display name |
| `tsconfigPath` | No | Path to `tsconfig.json` for JavaScript modules — required when using TypeScript in `src/server/` |

## JavaScript Modules (`src/server/`)

Reusable server-side logic lives in `src/server/` as JavaScript or TypeScript modules. These are **not** Fluent files — they are compiled separately and imported into scripts.

### Defining a module

```typescript
// src/server/handler.ts

import { API } from "@servicenow/glide";  // Glide API

export function process(request: any, response: any) {
  response.setBody({ message: 'Hello' })
}
```

> **TypeScript import note (src/server/ only):** When importing from one `.ts` file to another within `src/server/`, you must include the `.ts` extension due to `allowImportingTsExtensions: true` in `src/server/tsconfig.json`.

### Using a module in a `.now.ts` script field

**Option 1 — Direct function reference:**
```typescript
import { process } from '../server/handler.ts'

RestApi({
  routes: [{ script: process }]
})
```

**Option 2 — Inline require (when the script must be a string):**
```typescript
script: `
  const { process } = require('dist/modules/server/handler.js')
  process(request, response)
`
```

> `require` paths reference compiled output at `dist/modules/server/`, not source `src/server/`.

## tsconfig.json (root — for Fluent files)

```json
{
  "compilerOptions": {
    "target": "ES2019",
    "module": "commonjs",
    "strict": true,
    "types": ["@servicenow/sdk/types"]
  },
  "include": ["src/**/*.now.ts"]
}
```

## src/server/tsconfig.json (for TypeScript modules)

```json
{
  "compilerOptions": {
    "rootDir": "/",
    "module": "es2022",
    "target": "es2022",
    "moduleResolution": "bundler",
    "allowJs": true,
    "declaration": false,
    "sourceMap": false,
    "skipLibCheck": true,
    "allowImportingTsExtensions": true,
    "noEmit": true
  },
  "include": ["./**/*.ts", "../../@types/**/*.modules.d.ts"],
  "exclude": ["**/*.now.ts"]
}
```

Register this in `now.config.json` via the `tsconfigPath` field.

## generated/ directory

The `src/fluent/generated/` directory is created and maintained by `now-sdk build`. **Never edit files in this directory** — changes will be overwritten on the next build.

## Monorepo (pnpm workspace)

For projects with multiple applications:

```yaml
# pnpm-workspace.yaml
packages:
  - 'apps/*'
```

Each application is a separate directory with its own `now.config.json` and `package.json`.
